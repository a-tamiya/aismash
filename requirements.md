# プロンプトファイターズ 要件定義書

## 1. プロジェクト概要

### 1.1 タイトル

**プロンプトファイターズ**

### 1.2 概要

本プロジェクトは、Unityで開発する**1vs1ローカル対戦型の2D格闘ゲーム**である。

プレイヤーは試合前に、キャラクター名と特徴を自由入力する。AIはその入力をもとに、キャラクターの見た目、4種類の技、技名、技の構造、技パラメータを生成する。生成されたキャラクター同士が、地続きの2DステージでHPを削り合う。

本作の中心体験は、**「自分の考えたキャラクターがAIによってゲーム内ファイターとして形になり、その場で対戦できること」**である。

### 1.3 想定利用シーン

本作は、主に**オープンキャンパスの出し物**として展示することを想定する。

想定プレイヤーは高校生や来場者であり、短時間で直感的に遊べることを重視する。

### 1.4 開発エンジン

- Unity
- 2Dプロジェクト
- C#

## 2. ゲームコンセプト

### 2.1 コンセプト文

> 言葉からファイターを生み出せ。  
> 名前と特徴を入力すると、AIが見た目と技を生成。  
> 世界に一体だけのキャラクターで、友達と戦う2D格闘ゲーム。

### 2.2 企画の核

本作では、プレイヤーがキャラクターを直接デザインするのではなく、**キャラクター名と特徴をAIに渡し、AIがそれを解釈してファイター化する**。

例：

```text
キャラクター名：シャドウキャット
特徴：黒い炎をまとった素早い猫の剣士。近距離で連続攻撃し、相手を怯ませる戦い方が得意。
```

AI生成例：

```text
見た目：黒い炎をまとった猫耳の剣士。細身で素早そうな2Dアニメ立ち絵。
近距離技：黒炎みだれ斬り
遠距離技：影火球
特殊技：闇猫ステップ
必殺技：ナイトメアラッシュ
```

このように、プレイヤーが考えるのは**キャラクター名と特徴**であり、AIがそこから**見た目・技名・技構造・技パラメータ**を生成する。

## 3. 対象ユーザー

### 3.1 メイン対象

- オープンキャンパスに来場する高校生
- AIやゲーム制作に興味のある来場者
- 短時間で遊べる対戦ゲームを求める人

### 3.2 体験時間

1プレイあたりの目安は以下とする。

```text
キャラ入力：1〜2分
AI生成待ち：数十秒〜1分程度を許容
技確認：30秒程度
対戦：60秒以内
リザルト確認：30秒程度
```

合計で、**1組あたり3〜5分程度**の体験を目指す。

## 4. ゲーム全体フロー

### 4.1 基本フロー

```text
1. タイトル画面
2. 1P キャラクター入力
3. 2P キャラクター入力
4. AI生成中画面
5. 生成待ち中のトレーニングモード（任意）
6. キャラクター・技確認画面
7. 対戦開始
8. 試合終了
9. AIコメント生成中画面
10. リザルト画面
11. タイトルへ戻る
```

### 4.2 画面一覧

| 画面 | 役割 |
|---|---|
| タイトル画面 | 最初に表示するゲーム開始画面。豪華でスタイリッシュなビジュアルで、ゲーム開始へ誘導する |
| キャラ入力画面 | 1P/2Pがキャラ名と特徴を入力 |
| 生成中画面 | AIがキャラ・画像・技を生成していることを表示 |
| トレーニング画面 | AI生成待ち中にテストキャラで移動・ガード・技の操作確認を行う |
| 技確認画面 | 生成されたキャラ画像、技名、説明、操作を確認 |
| 対戦画面 | 1vs1バトルを行う |
| AIコメント生成中画面 | 試合ログをもとにAIコメントを生成 |
| リザルト画面 | 勝者、キャラ情報、試合結果、AIコメントを表示 |

## 5. キャラクター生成仕様

### 5.1 プレイヤー入力

各プレイヤーは、試合前に以下を入力する。

| 項目 | 内容 | 必須 |
|---|---|---|
| キャラクター名 | プレイヤーが考えた名前 | 必須 |
| 特徴 | 見た目、性格、戦闘スタイル、属性、モチーフなど | 必須 |

### 5.2 入力例

```text
キャラクター名：雷神ペンギン
特徴：雷をまとったペンギンの戦士。小柄だが素早く、遠距離から雷を飛ばしつつ、近づくと高速連続攻撃をする。
```

```text
キャラクター名：紅蓮の侍ドラゴン
特徴：炎を操るドラゴンの侍。大きな刀を持ち、攻撃は遅いが一撃が重い。必殺技は炎の突進斬り。
```

```text
キャラクター名：もちもちスライム忍者
特徴：かわいいスライムの忍者。分身や煙幕で相手を惑わせ、素早く近づいて攻撃する。
```

### 5.3 AIが生成する内容

AIは、各プレイヤーの入力から以下を生成する。

| 生成項目 | 内容 |
|---|---|
| キャラクター見た目説明 | 2Dアニメ立ち絵としての外見説明 |
| 画像生成用プロンプト | キャラクター画像生成に使用するプロンプト |
| 技4種類 | 近距離技、遠距離技、特殊技、必殺技 |
| 技名 | 各技の名前 |
| 技説明 | プレイヤーに表示する説明文 |
| 技構造 | Unityが実行するアクション列 |
| 技パラメータ | ダメージ、射程、発生、後隙、クールダウンなど |
| 属性 | 技の属性 |
| 状態異常 | 技が付与する状態異常 |
| 危険度 | 技の強さ・扱いづらさを示す指標 |

### 5.4 キャラクター画像仕様

| 項目 | 内容 |
|---|---|
| 画像形式 | 2Dアニメ立ち絵 |
| 枚数 | 1キャラにつき1枚 |
| 戦闘中の扱い | 立ち絵1枚を移動・揺れ・拡大縮小・残像・エフェクトで動かす |
| アニメーション | 歩き・攻撃・ジャンプなどの個別アニメーションはMVPでは生成しない |
| 背景 | 可能であれば透過。難しい場合は単色背景またはカード風表示で対応 |

## 6. 技生成仕様

### 6.1 技枠

各キャラクターは、AIによって4種類の技を持つ。

| 技枠 | 役割 | 例 |
|---|---|---|
| 近距離技 | 近くの相手を攻撃する基本技 | みだれひっかき、居合斬り、連続パンチ |
| 遠距離技 | 離れた相手に攻撃する技 | 毛玉ショット、火球、雷弾 |
| 特殊技 | 移動、妨害、防御、状態異常などの個性技 | 猫だまし、煙幕、カウンター、引き寄せ |
| 必殺技 | 高威力・高演出の大技 | にゃんこラッシュ、紅蓮突進、雷神連撃 |

技枠は固定するが、**技名・内容・パラメータはキャラクターごとにAIが生成する**。

### 6.2 技生成の基本方針

固定技を事前に用意するのではなく、Unity側に「技を構成する基本部品」を実装し、AIがそれらを組み合わせて技レシピを生成する。

例：

```text
プレイヤー入力：
黒い炎をまとった猫の剣士

AI生成：
近距離技：黒炎みだれ斬り
構造：melee_hitbox → multi_hit → apply_status(stun)
パラメータ：3ヒット、短射程、低ダメージ、短い怯み
```

### 6.3 技構成部品

MVPでUnity側に実装する技部品は以下とする。

| 部品ID | 内容 |
|---|---|
| `melee_hitbox` | 近距離攻撃判定を出す |
| `projectile` | 飛び道具を発射する |
| `area_hitbox` | 円形・矩形などの範囲攻撃を出す |
| `dash` | 前方または後方に素早く移動する |
| `jump_attack` | 空中または上方向に攻撃する |
| `counter` | 一定時間、相手の攻撃に反応して反撃する |
| `buff_self` | 一定時間、自分を強化する |
| `pull_enemy` | 相手を自分側または中心へ引き寄せる |
| `push_enemy` | 相手を押し出す、吹き飛ばす |
| `apply_status` | 状態異常を付与する |
| `delay` | 次のアクションまで時間差を作る |
| `multi_hit` | 複数回ヒットする攻撃にする |

### 6.4 状態異常

MVPで実装する状態異常は以下とする。

| 状態異常ID | 内容 |
|---|---|
| `stun` | 短時間怯ませる。操作不能時間は短めに制限する |
| `burn` | 数秒間、継続ダメージを与える |
| `slow` | 一定時間、移動速度を下げる |
| `guard_break` | ガード状態を崩し、短時間無防備にする |

### 6.5 属性

技には属性を持たせる。属性は主に演出と軽い効果に使用する。

| 属性ID | 内容 |
|---|---|
| `none` | 無属性 |
| `physical` | 物理 |
| `fire` | 炎。burnと相性が良い |
| `ice` | 氷。slowと相性が良い |
| `lightning` | 雷。stunと相性が良い |
| `dark` | 闇。pullやguard_breakと相性が良い |
| `wind` | 風。pushやdashと相性が良い |

## 7. 技パラメータ仕様

### 7.1 共通パラメータ

各技は、最低限以下のパラメータを持つ。

| パラメータ | 内容 |
|---|---|
| `damage` | 基本ダメージ |
| `range` | 射程 |
| `startup` | 発生までの時間 |
| `active_time` | 攻撃判定の持続時間 |
| `recovery` | 技後の硬直時間 |
| `cooldown` | 再使用可能になるまでの時間 |
| `hit_count` | ヒット回数 |
| `knockback` | 吹き飛ばし量 |
| `stun_time` | 怯み時間 |
| `guard_damage` | ガード時に削る量 |
| `move_force` | 技による移動量 |
| `element` | 属性 |
| `status_effect` | 付与する状態異常 |
| `risk_level` | 技の危険度 |

### 7.2 HP・ダメージ基準

MVPでは以下を基準とする。

| 項目 | 値 |
|---|---|
| HP | 100 |
| 近距離技ダメージ | 8〜14 |
| 遠距離技ダメージ | 6〜12 |
| 特殊技ダメージ | 4〜10＋状態異常 |
| 必殺技ダメージ | 18〜30 |
| 決着目安 | 8〜12回程度の有効ヒットで決着 |

### 7.3 技ごとの目安

| 技枠 | ダメージ | クールダウン | 特徴 |
|---|---:|---:|---|
| 近距離技 | 8〜14 | 1.0〜2.5秒 | 当てやすいが射程短め |
| 遠距離技 | 6〜12 | 2.0〜4.0秒 | 離れて攻撃できるが後隙あり |
| 特殊技 | 4〜10 | 4.0〜7.0秒 | 状態異常・移動・妨害など |
| 必殺技 | 18〜30 | 8.0〜15.0秒 | 高威力だが発生・後隙が大きい |

### 7.4 MP・ゲージ

MVPでは、**MPやゲージ消費は実装しない**。

技の使用制限は、主に以下で管理する。

- クールダウン
- 発生速度
- 後隙
- 射程
- リスクレベル

## 8. AI生成結果の補正仕様

### 8.1 基本方針

プレイヤーは強そうなキャラクターを自由に作ってよい。  
ただし、AIやUnity側は必ずバランス補正を行う。

例：

```text
入力：全てを一撃で破壊する無敵の神

補正後：
高火力の必殺技を持つが、発生が遅く、後隙とクールダウンが非常に長いキャラクター
```

### 8.2 強すぎるプロンプトの扱い

以下の方針とする。

- 強そうなプロンプトは歓迎する
- ただし、ゲーム内では必ずバランス補正する
- 強すぎる表現がある場合、注意表示を出す
- 「即死」「必中」「完全無敵」などは、そのまま実装せず、リスクや制限付き効果に変換する

### 8.3 注意表示例

```text
このキャラクターは非常に強力な特徴を持っています。
ゲームバランスを保つため、技には長いクールダウンや大きな後隙が付与されます。
```

### 8.4 補正ルール

Unity側はAI出力をそのまま採用せず、以下の制限をかける。

| 項目 | 制限 |
|---|---|
| 最大ダメージ | 通常技は14、特殊技は10、必殺技は30を目安に制限 |
| 最小クールダウン | 技枠ごとに下限を設定 |
| 最大射程 | ステージ幅に対して過剰にならないよう制限 |
| 最大怯み時間 | 長時間の操作不能を禁止 |
| 無敵 | MVPでは原則禁止。必要な場合は短時間かつ高リスク扱い |
| ガード不能 | 原則禁止。guard_breakとして短時間効果に変換 |
| 多段ヒット | 総ダメージが上限を超えないよう各ヒットを調整 |

### 8.5 リスクレベル

技には危険度を持たせる。

| リスク | 内容 |
|---|---|
| `low` | 扱いやすいが威力は低め |
| `medium` | 標準的な性能 |
| `high` | 強いが発生・後隙・クールダウンが大きい |
| `extreme` | 非常に強いが外すと大きな不利を背負う |

## 9. バトル仕様

### 9.1 対戦形式

| 項目 | 内容 |
|---|---|
| 対戦形式 | プレイヤー vs プレイヤー |
| 対戦方式 | ローカル対戦 |
| 勝利条件 | 相手のHPを0にする |
| 時間制限 | 60秒 |
| ラウンド | 1本勝負 |
| タイムアップ | 残りHPが多い方の勝利 |
| ステージ | 地続きの横長ステージ |
| 場外撃墜 | なし |
| 画面端 | あり |

### 9.2 キャラクター基本アクション

| アクション | 内容 |
|---|---|
| 左右移動 | 地上を左右に移動する |
| ジャンプ | 上方向にジャンプする |
| ガード | 前方からのダメージを軽減する |
| 近距離技 | 技1 |
| 遠距離技 | 技2 |
| 特殊技 | 技3 |
| 必殺技 | 技4 |

### 9.3 ノックバック

- 攻撃命中時、相手を一定距離押し出す
- 必殺技や強い技ほどノックバックが大きい
- 画面端では、ノックバックによって壁に押し付けられる
- 壁バウンドはMVPでは必須ではない

### 9.4 ガード

| 項目 | 内容 |
|---|---|
| ガード方式 | ボタン式ガード |
| ガード効果 | 被ダメージ軽減 |
| ガード削り | 技によって少量のガードダメージあり |
| ガード崩し | 状態異常 `guard_break` により短時間ガード不能 |

### 9.5 画面端

- ステージ左右端に壁を配置する
- プレイヤーは画面外に出ない
- 画面端では逃げ場が少なくなるため、ノックバックやガード崩しが重要になる

## 10. 操作仕様

### 10.1 コントローラー対応

オープンキャンパス展示を想定し、**ゲームパッド2個でのローカル対戦対応を優先**する。

推奨操作：

| 操作 | コントローラー |
|---|---|
| 移動 | 左スティック / 十字キー |
| ジャンプ | A / Cross |

| 遠距離技 | Y / Triangle |
| 特殊技 | B / Circle |
| 必殺技 | RB / R1 |
| 決定 | A / Cross |
| キャンセル | B / Circle |

ゲームパッドが使用できない場合に備えて、キーボード2人操作にも対応する。

#### 1P

| 右移動 | D |
| ジャンプ | W |
| ガード | Left Shift |
| 近距離技 | J |
| 遠距離技 | K |
| 特殊技 | L |
| 必殺技 | I |

#### 2P

| 操作 | キー |

## 11. 演出仕様

### 11.1 立ち絵の動かし方

キャラクター画像は1枚のみ生成し、以下の演出で動きを表現する。

| 状態 | 演出 |
|---|---|
| 待機 | 軽い上下揺れ |
| 移動 | 左右移動＋少し傾き |
| ジャンプ | 上昇・落下に合わせて傾き |
| 攻撃 | 前方に一瞬踏み込む、拡大、残像 |
| 被弾 | 画面揺れ、点滅、後退 |
| ガード | 防御エフェクト表示 |
| 必殺技 | カメラ揺れ、強いエフェクト、スローモーション風演出 |

### 11.2 エフェクト

技の見た目は、AIの属性や技構造に応じてUnity側の汎用エフェクトを組み合わせる。

| 属性 | エフェクト例 |
|---|---|
| fire | 炎、赤系、爆発 |
| ice | 氷、青系、結晶 |
| lightning | 雷、黄色系、電撃 |
| dark | 黒紫系、影、吸い込み |
| wind | 緑系、風切り、吹き飛ばし |
| physical | 白・灰色系、衝撃波 |

## 12. AI連携仕様

### 12.1 AIを使用するタイミング

AIを使用するタイミングは以下とする。

| タイミング | 内容 |
|---|---|
| 試合前 | キャラクター見た目、画像、技4つを生成 |
| 試合後 | 試合ログをもとにAIコメントを生成 |

対戦中はAIに問い合わせない。  
そのため、バトル中の遅延や通信待ちは発生しない。

### 12.2 AI生成の待ち時間

試合前のみ生成するため、生成待ち時間は許容する。

生成中画面では以下を表示する。

```text
AIがキャラクターと技を生成中...
しばらくお待ちください。
```

生成待ち中は、プレイヤーが任意でトレーニングモードに入れるようにする。

トレーニングモードでは以下を満たす。

- API生成結果ではなく、事前に用意したテストキャラ・サンプル技・仮キャラ画像を使用する
- 1P/2Pの移動、ジャンプ、ガード、4技の入力確認ができる
- 時間制限と勝敗判定は行わない
- HPが0になった場合は短時間後に位置・HP・クールダウンをリセットする
- 生成完了またはプレイヤー操作により、キャラクター確認画面またはキャラ選択画面へ戻れる

### 12.3 キャラクター生成JSON

AIは、Unityが読み取りやすいJSON形式で出力する。

例：

```json
{
  "character_name": "シャドウキャット",
  "input_features": "黒い炎をまとった素早い猫の剣士。近距離で連続攻撃する。",
  "visual_prompt": "2D anime standing character, black cat swordsman, dark flames, cool and agile, full body, game character art",
  "visual_description": "黒い炎をまとった猫耳の剣士。細身で素早そうなシルエット。",
  "skills": [
    {
      "slot": "close",
      "skill_name": "黒炎みだれ斬り",
      "description": "黒い炎をまとった爪と剣で素早く3回斬りつける近距離技。",
      "element": "fire",
      "risk_level": "medium",
      "parameters": {
```

### 12.4 AIプロバイダー方針

AIの利用形態は、開発フェーズと本番運用で段階的に切り替える。

#### 開発中（ローカルLLM）

| 項目 | 内容 |
|---|---|
| 使用モデル | Qwen2.5 VL（予定） |
| 実行環境 | Ollama（ローカル） |
| エンドポイント | `http://localhost:11434` |
| 用途 | キャラクター・技パラメーター生成、試合後コメント生成 |
| 画像生成 | 開発中はスキップ、またはサンプル画像で代替 |

ローカルLLMを使う理由は、API費用なしに高速な開発サイクルを回せるためである。

#### 本番・展示時（OpenAI API）

| 項目 | 内容 |
|---|---|
| テキスト生成 | OpenAI Chat Completions API（GPT-4o等） |
| 画像生成 | OpenAI Image Generation API（DALL·E 3等） |
| 用途 | キャラクター見た目・技パラメーター生成、画像生成、試合後コメント生成 |

OpenAI API切り替えのポイント：

- テキスト生成（技パラメーター・コメント）はChat Completions APIで行う
- 画像生成は `visual_prompt` フィールドをそのまま DALL·E に渡す
- Unity側のAPIクライアントはエンドポイントとモデル名を設定で切り替えられる構造にする
- APIキーはUnityの `StreamingAssets` 以外の安全な場所、またはビルド時の環境変数で管理する

#### 切り替え方針

```text
開発中：Ollama（Qwen2.5 VL） → 画像なし or サンプル画像
展示前：OpenAI API に切り替え → DALL·E で画像生成も有効化
```

Unityのコード側は、プロバイダーを抽象化したインターフェースを用意し、設定ファイルまたはインスペクターで切り替えできるようにする。

## 13. 保存仕様

### 13.1 保存対象

生成したキャラクターは、次回以降も使えるように保存する。

保存対象は以下とする。

| 項目 | 保存するか |
|---|---|
| キャラクター画像 | 保存する |
| キャラクター名 | 保存する |
| 入力された特徴プロンプト | 保存する |
| AI生成された技JSON | 保存する |
| 画像生成用プロンプト | 保存する |
| 生成日時 | 保存する |

### 13.2 保存形式

MVPではローカルファイル保存を想定する。

```text
/SaveData/Characters/{character_id}/character.json
/SaveData/Characters/{character_id}/character.png
```

### 13.3 キャラクター再利用

保存したキャラクターは、タイトル画面またはキャラ入力画面から呼び出せるようにする。

MVPでは、以下の機能を目標とする。

- 保存済みキャラ一覧を表示
- 保存済みキャラを1Pまたは2Pに読み込む
- 読み込んだキャラで再戦可能にする

## 14. リザルト仕様

### 14.1 表示内容

リザルト画面では以下を表示する。

| 項目 | 内容 |
|---|---|
| 勝者 | 1Pまたは2P |
| 勝者キャラ画像 | 勝者の生成画像 |
| 敗者キャラ画像 | 敗者の生成画像 |
| キャラクター名 | 1P/2Pの名前 |
| 使用プロンプト | 1P/2Pが入力した特徴 |
| 残りHP | 試合終了時のHP |
| 一番使った技 | 各プレイヤーが最も多く使用した技 |
| 一番ダメージを与えた技 | 各プレイヤーの最大貢献技 |
| AI試合コメント | 試合内容に対する短いコメント |

### 14.2 AIコメント生成演出

試合終了後、リザルトの前に短い生成中演出を入れる。

```text
AIが試合を分析中...
```

AIコメントが生成できたらリザルト画面に表示する。

### 14.3 AIコメント失敗時

AIコメント生成に失敗した場合は、Unity側のテンプレコメントを表示する。

例：

```text
接戦でした。勝者は自分の技の特徴をうまく活かして勝利しました。
```

## 15. 生成失敗時の保険仕様

オープンキャンパス展示では、通信やAPIの不具合が発生してもゲームを続行できることが重要である。

### 15.1 画像生成失敗時

画像生成に失敗した場合は、以下の順で対応する。

```text
1. 再生成ボタンを表示
2. それでも失敗した場合、事前に用意した仮キャラ画像を使用
3. キャラクター名と技はそのまま使用
```

### 15.2 技生成失敗時

技生成に失敗した場合は、以下の順で対応する。

```text
1. 再生成ボタンを表示
2. 再生成に失敗した場合、サンプル技セットを読み込む
3. サンプル技セットには、近距離・遠距離・特殊・必殺の4技を必ず含める
```

### 15.3 オフライン展示モード

APIが使えない状況に備え、事前生成済みのサンプルキャラクターを複数保存しておく。

オフライン展示モードでは、以下ができるようにする。

- サンプルキャラを選んで対戦
- 保存済みキャラを選んで対戦
- AI生成なしでデモを継続

### 15.4 API準備中の開発・展示モード

OpenAI APIキーや画像生成APIが未準備の場合でも、ゲーム本体の操作確認を進められるようにする。

- 試合前UIからトレーニングモードを開始できる
- トレーニングモードは `StreamingAssets/Presets` のサンプルキャラと `StreamingAssets/Sprites/test.jpg` を使用する
- AI生成中画面の代替として、来場者や開発者が待ち時間に操作を練習できる
- API連携が有効になった後も、生成待ち中の待機体験として残す

## 16. 安全性・表示内容の制限

オープンキャンパスで高校生が遊ぶことを想定し、不適切な入力や生成結果を避ける。

### 16.1 入力制限

以下の内容を含む入力は、警告または再入力対象とする。

- 過度に暴力的な表現
- 性的な表現
- 差別的な表現
- 実在人物への攻撃的表現
- 個人情報
- 公序良俗に反する内容

### 16.2 生成結果の制限

AI生成結果に不適切な表現が含まれる場合、以下の対応を行う。

```text
1. 表示前に検査
2. 問題がある場合は再生成
3. 再生成に失敗した場合は安全なサンプル表現に置き換え
```

## 19. 受け入れ条件

MVP完成とみなす条件は以下とする。

1. 1Pと2Pがそれぞれキャラクター名と特徴を入力できる
2. AIが各キャラクターの画像と4つの技を生成できる
3. 生成された技がUnity上で実行できる
4. 1vs1のローカル対戦が成立する
5. HPを0にする、または60秒経過で勝敗が決まる
6. ガード、ジャンプ、ノックバック、画面端が機能する
7. 状態異常が最低4種類機能する
8. 技JSONとキャラクター画像を保存できる
9. 保存済みキャラクターを再利用できる
10. 試合後にリザルトとAIコメントを表示できる
11. AI生成に失敗してもサンプルキャラでデモを継続できる
